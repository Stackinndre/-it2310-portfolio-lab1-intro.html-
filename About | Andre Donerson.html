Project Reflection

What I Learned
Building this portfolio from scratch taught me how much the three core web technologies depend on each other. HTML gives you structure, but it's lifeless without CSS. CSS makes things look right, but without JavaScript nothing responds. Wiring all three together — scroll reveals, form validation, animated loaders, skill bars — showed me how real production sites actually work under the hood. I also learned that planning the design first saves far more time than fixing layout issues after the fact.
Were the Results What I Expected?
Mostly, yes — but the JavaScript was harder than I anticipated. Getting the IntersectionObserver to trigger at exactly the right scroll position, and keeping the mobile nav accessible while also animating correctly, took significantly more iteration than the HTML and CSS work did. The final result actually exceeded my original plan: I ended up building two resume versions, a portfolio filter, a custom loader, a scroll progress bar, and a typewriter effect — all in vanilla JS with no libraries.
Key Findings & Takeaways
1. Semantic HTML matters. Using the right tags from the start made CSS targeting and JavaScript queries much simpler.

2. CSS custom properties are powerful. Defining color and font tokens in :root made global changes a one-line edit.

3. Test on mobile early. Every layout I built for desktop needed adjustments for smaller screens — doing this at the end was painful.

4. JavaScript can do a lot without a framework. Everything on this site — animations, validation, filtering — is plain ES5 JS. Frameworks solve real problems, but you need to understand the fundamentals first.
